## [0.0.86] - 2025-10-20

### Added
- New `rumdl.format.autoFixOnSave` configuration option to automatically fix markdown issues when saving files
- Proper TOML parser (`@iarna/toml`) for robust configuration validation
- Integration tests for config validation and pyproject.toml extraction
- Two-phase validation system (TOML syntax validation â†’ semantic validation)

### Fixed
- **Issue #63**: Fixed incorrect handling of multiline indented arrays in config files by replacing fragile regex-based parsing with proper TOML parser
- Fixed pyproject.toml section conversion bug: `[tool.rumdl]` now correctly converts to `[global]` instead of `[rules]`
- Fixed pyproject.toml rule sections: `[tool.rumdl.MD013]` now correctly converts to `[rules.MD013]`
- Fixed formatting test by providing required FormattingOptions (tabSize)
- Made Logger defensive to handle uninitialized state in test environment, improving test reliability

### Changed
- Config validator now validates the actual default config generated by `rumdl init`
- Config validator now properly handles multiline arrays in both .rumdl.toml and pyproject.toml files
- Improved error messages with accurate line numbers for TOML syntax errors
- All 88 tests passing with comprehensive coverage

### Technical
- Replaced line-by-line regex parsing with structured TOML AST parsing
- Separated syntax validation from semantic validation for better error reporting
- Added `ConfigValidator.validateToml()` methods for working with parsed TOML values
- Implemented `onWillSaveTextDocument` handler for autoFixOnSave functionality
